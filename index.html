DNSを変更するにあたり独自ドメイン宛てのメールに対して自動で変更通知する

ドメイン管理を valueドメインから Route53 に変更する。
それにあたってのひとまずメモメモ。

変更にあたり、valueドメインの設定で独自ドメイン宛てのメール（アドレス1つだけ）をgmailに転送していたので、これをどうにかする。

今後この独自ドメインのメールアドレスは使わずGmailのアドレスに一本化する。
なので、独自ドメインにきたメールに対してはGmailのアドレスを通知する感じで自動返信を行いたい。

Amazon WorkMail

Amazon WorkMail の Auto Responseがつかえそうな、、、気がする。

Amazon WorkMail （セキュリティに優れたマネージド型の企業向けE メールおよびカレンダーサービス） | AWS
https://aws.amazon.com/jp/workmail/

SES でもメールの受信を出来るようになったみたいなのだが、そこから色々やるのはちょっと手間そう。
Amazon WorkMail でうまくできなかったらその方向。

とりあえず、$4らしいので、1年か2年くらい運用したら止める。

Drupal

DrupalのURLが新しいものにならないときはURLエイリアスを削除する

Drupal ですでに登録してあるコンテンツのURLエイリアスを変更しても、元のもののままだったりしてしまう場合は

環境設定　> 検索とメタデータ > URL エイリアス

といくと、 URLエイリアスの一覧がでてくるので、そこで古いものを削除すると、新しいものになる

DeployHQ のユーザの種類メモ

DeployHQ のユーザの種類メモ

Account administration - access to all aspects of the account including billing
User management - access to all projects and user management
All projects - access to all projects with the ability to make all changes including deleting a project
Specific projects - access to a certain project with restricted abilities, i.e. a user cannot delete a project

Getting Started - DeployHQ
https://www.deployhq.com/academy/manual/getting-started

  • Account administration - とりあえずアドミン
  • User management - 全部のプロジェクトの操作とユーザー管理
  • All projects - 全部のプロジェクトにアクセスできて、削除も可能
  • Specific projects - 特定のプロジェクトのみ

Fastly memo

Fastly
https://www.fastly.com/

キャッシュの設定がそれぞれ変更される都度version管理されている

VCL（Varnish Configuration Language）というのに記載する。

Authenticating before returning a request - VCL | Fastly Help Guides
https://docs.fastly.com/guides/vcl/authenticating-before-returning-a-request

こんなかんじでかけばキャッシュさせないURLの設定とかできる様子。

特定のリクエストをキャッシュ対象外に設定する - Japanese - Fastly’s Community
https://community.fastly.com/t/topic/875

Cache を返しているかどうかの確認

キャッシュ関係のヘッダは「Cache-Control」「Expires」「Pragma」。

ブラウザのdeveloper tools とかで Network で該当ページのHTMLなりを選択して、Response headers を確認する。

キャッシュヒットしてるとこの辺で返ってくる。

FastlyにキャッシュされたオブジェクトのTTL確認方法 - Japanese - Fastly’s Community
https://community.fastly.com/t/fastly-ttl/867
