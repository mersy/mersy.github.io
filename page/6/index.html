iPadでのブログ原稿ツールにBywordを使ってみる

ちょっと延々と電車に乗ってる時にこのブログの原稿を描きたいなぁと思うことがあり色々と調べていた。

とりあえず、iPadで開いたので、Dropbox にある Hexo の原稿の Markdown の原稿データをいじりやすいアプリはないものか？ということを延々と探してた。

iPad でも Dropbox の同期が使えるアプリはあるのだけど、専用のディレクトリを切るとかだったりでそれはすでにあるディレクトリのと対応が良くないので、無しということで。

今は、PC で作業するときは Dropbox のディレクトリ内に hexo 用のディレクトリを切ってそこで作業してるので、そのディレクトリにあるmdファイルを操作しやすい、という前提でアプリを探してみた。

boostnoteとかも良さそうではあったのだけど、専用のディレクトリを切るっぽいので今回はなしってことで、ひとまず　Byline　に落ち着いた

Boostnote | Boost Happiness, Productivity, and Creativity.
https://boostnote.io/

iAwriter が良さそうな気がしたのだけど、Dropboxでデータが消える、みたいなレビューをみたのでちょっとそれは地獄なので、、、と。。。

iA
https://ia.net/writer/

色々みてたら、 Byword に落ち着いた。

Byword - Markdown text editor app for Mac, iPhone and iPad.
https://bywordapp.com/

既存であるものだと formatter が書かれてあるので楽だけど、新規だとそうもいかない気もするな。。。

テンプレート的なものがないかどうか、その辺をみてもいいのかもな。

Hexo を使ってる時点で Publish は PC 前提になるのだけど、そこはまぁいいかなー、という気がする。

iPad でのキーボードも考えた方が良さそうだ。


ちなみに、markdown でのメモツールとしては Bear を今はメインで使っている。
Mac と iPhone で。

Bear - Notes for iPhone, iPad and Mac
http://www.bear-writer.com/

これでしばらくはいいかなー、というところ。

ただ、ブログを書く用途とは別なので、そこは混在させたくないな、となんとなく思ったので、アプリレベルで別にしておこうかな、と。
できるかどうかわからないけど。。。

CMS実装のための仕様共有方法を考える

ワイヤーにコメントをつける感じのくらいが個人的にはベストだと思っているのだけれども。

開発者のやりとりも捗る！プロトタイピングツール「InVision」って何？ - WPJ
https://www.webprofessional.jp/evolution-of-prototyping-tool-invision/

そう考えるとPowerPointが今のところはまともなツールのようなきがしてしまう。

いいと言えばいいのだけど、PowerPointはワイヤーを作る方の機能が若干足りない気がする。

  • DIST.18 「Sketch時代のWebデザインワークフロー」 - 勉強会メモ - Qiita
    https://qiita.com/ezawa800/items/b465180208419a96662e
  • marcosvidal/Sketch-Notebook
    https://github.com/marcosvidal/Sketch-Notebook

この辺を読んでみて、とりあえず Sketch でやって見るのが良さそうな気がしている。
他のアプリケーションもいずれは似たような機能ができるようになるとは思うのだけど。

Mac しかないみたいだけど、その辺は Invision あたりを経由させるとかでどうにかしたいところではある。

このへんは案件によっても異なってくるだろうし、どういうデザインデータを共有されるか、とかでも変わるだろうし、どのスコープ、タイミングから入ってくるかも変わるだろうから一概には言えないだけに難しいところだなー。

とりあえず、なんとなくなワイヤーとpsdだけが来てしまって、さて、CMSに組み込む際に仕様をどこかにまとめておかねば、というタイミングで、markdownのテキストにまとめるだけで、実装する側が迷わずにいけるのかどうか？この点を重視しないといけないなぁ、と思っているところである。

とは言え、テキストレベルですら仕様をまとめきれていないこともあり、メンバーに迷惑をかけているのでそれはそれとして個人的な課題であるので改善しないといけないな。。。

前者はPCとかで作業してるときはその方が楽そうだし、後者はブラウザからやるときは楽そうだな、と。
netlify cms は iPad で日本語入力が出来なかった。。。それだけが残念すぎる。

サーバに静的ジェネレータおいといてそこで生成させるとかでも出来そうな気がするけど。
MTとDataAPIでもいいわけだし。

contentfulとかnetlify cmsとかみてて大体よさそうなのだけど最後なんかちょっとたりないな、、と。
ないなら作ればいいのかもしれないけど。

はてなブログと同期するツールがあったのだけどこれがなかなかよさそうなかんじではあるけど、鬼門はやはり画像か。

  • motemen/blogsync: Push and pull blog entries from/to local filesystem
    https://github.com/motemen/blogsync

memo netlify cms

Netlify CMS | Open-Source Content Management System
https://www.netlifycms.org/docs/start-with-a-template/

CMSっぽいものがあるみたいなので試してみた。

githubのリポジトリと連携。

Static Site Generator については GatsbyJS を選択してみた。Hugoも選択可能。

GatsbyJS
https://www.gatsbyjs.org/

ボタンをポチるだけで問題なくdeployまでできてすげー、と。

/admin/ で管理画面ぽいのもあるのでコレは便利そう。

最初、管理画面から記事を投稿した際にpathの指定を / からはじめなかったからコンパイルエラーになっていた。

原稿のmdファイルもmasterにコミットすれば反映された。
これは結構理想に近いかもしれない。

アイテム周りの置き場所がいまいちわかっていないのでその辺は整理してみる。

Home | Gatsby + Netlify CMS
https://mersy.netlify.com/

できれば見た目の選択が簡単にできて、あとは原稿とアイテムを管理画面、ローカルからのコミットの両方が出来れば結構いい感じかも。

下書きっていう概念がないのかな。その辺はもう少しドキュメントを読んでみようと思う。

もし問題なく使えるようであれば、サブドメインを設定しつつ、githubpagesにおいてるものを持って行ってもいいのかもしれない。
pathの追加が必要ってことだろうなー。
