ローカルに保存したHTMLファイルのPVをGoogle Analyticsで計測するための設定

HTMLファイルをローカルに保存してもらい、それをひらいたあとのイベント等を計測しようと思って色々調べたときのメモ。

https://stackoverflow.com/questions/3591847/google-analytics-from-a-file-url
https://stackoverflow.com/questions/22761200/using-google-analytics-without-a-sever-or-localhost

この辺を見る限り

ga('create', 'UA-xxxxxxxx-x',{'cookieDomain': 'none'});
ga('set', 'checkProtocolTask', null);
ga('set', 'checkStorageTask',  null);
ga('set', 'historyImportTask',  null);
ga('send', 'pageview');

という感じで書けば良さそう。

あと、ページ開いたこと自体を計測する場合は

ga('set', 'page', '/dummy/index.html');

のようにURLもセットしておく必要があるっぽい。

DrupalのURLが新しいものにならないときはURLエイリアスを削除する

Drupal ですでに登録してあるコンテンツのURLエイリアスを変更しても、元のもののままだったりしてしまう場合は

環境設定　> 検索とメタデータ > URL エイリアス

といくと、 URLエイリアスの一覧がでてくるので、そこで古いものを削除すると、新しいものになる

DeployHQ のユーザの種類メモ

DeployHQ のユーザの種類メモ

Account administration - access to all aspects of the account including billing
User management - access to all projects and user management
All projects - access to all projects with the ability to make all changes including deleting a project
Specific projects - access to a certain project with restricted abilities, i.e. a user cannot delete a project

Getting Started - DeployHQ
https://www.deployhq.com/academy/manual/getting-started

  • Account administration - とりあえずアドミン
  • User management - 全部のプロジェクトの操作とユーザー管理
  • All projects - 全部のプロジェクトにアクセスできて、削除も可能
  • Specific projects - 特定のプロジェクトのみ

キャッシュの設定がそれぞれ変更される都度version管理されている

VCL（Varnish Configuration Language）というのに記載する。

sub vcl_recv {

  /* unset state tracking header to avoid client sending it */
  if (req.restarts == 0) {
    unset req.http.X-Authed;
  }

  if (!req.http.X-Authed) {
    /* stash the original URL and Host for later */
    set req.http.X-Orig-URL = req.url;

    /* set the URL to what the auth backend expects */
    set req.url = "/authenticate";

    /* Auth requests won't be cached, so pass */
    return(pass);
  }

  if (req.http.X-Authed == "true") {
    /* were authed, so proceed with the request */
    /* reset the URL */
    set req.url = req.http.X-Orig-URL;

  } else {
    /* the auth backend refused the request, so 403 the client */
    error 403;
  }

Authenticating before returning a request - VCL | Fastly Help Guides
https://docs.fastly.com/guides/vcl/authenticating-before-returning-a-request

if (req.url ~ "^/(hoge|fuga|login)/") {

}

Cache を返しているかどうかの確認

キャッシュ関係のヘッダは「Cache-Control」「Expires」「Pragma」。

ブラウザのdeveloper tools とかで Network で該当ページのHTMLなりを選択して、Response headers を確認する。

キャッシュヒットしてるとこの辺で返ってくる。

X-Cache-Hits : 1
X-Cache    : HIT

FastlyにキャッシュされたオブジェクトのTTL確認方法 - Japanese - Fastly’s Community
https://community.fastly.com/t/fastly-ttl/867

そのほか Response headers

Cache-Control    : no-store, no-cache, must-revalidate
Pragma : no-cache
